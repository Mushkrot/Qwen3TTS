# Dataset Contract (Sub-task #1)

## Goal

Define a strict, reproducible dataset format for Qwen3-TTS single-speaker fine-tuning:
- source voice: Russian recordings,
- target generation: English text,
- priority: speaker similarity and naturalness.

---

## 1) Required input fields

Each training row must include:

- `audio`: absolute path to WAV file
- `text`: transcript text for that WAV (UTF-8)
- `ref_audio`: absolute path to a reference WAV

JSONL example:

```jsonl
{"audio":"/data/ru_ds/utt0001.wav","text":"Это пример русской реплики.","ref_audio":"/data/ru_ds/ref.wav"}
{"audio":"/data/ru_ds/utt0002.wav","text":"She said she would be here by noon.","ref_audio":"/data/ru_ds/ref.wav"}
```

Recommendation:
- Use the same `ref_audio` for all rows to stabilize speaker identity.

---

## 1.1) Project-local folder layout

The Qwen3TTS dataset workspace lives inside this repository:

```text
datasets/
  voices/
    <VoiceName>/
      Input/
      Ready/
```

- `Input/`: raw source recordings for the voice.
- `Ready/`: processed builder outputs for that voice.
- `Input/` and `Ready/` contents are local working data and must not be committed.
  Only folder scaffolds (`.gitkeep`) and small README/policy files are tracked.

Current canonical voice folders:

```text
datasets/voices/Dima/Input
datasets/voices/Dima/Ready
datasets/voices/Baritone/Input
datasets/voices/Baritone/Ready
```

`/ai/whisper1` is a separate project and must not be treated as the Qwen3TTS
dataset workspace.

Current local source state on 2026-06-22:

- `datasets/voices/Baritone/Input/` contains three local MP3 source files.
- Those files are intentionally ignored by Git because the original source audio is stored outside this repository.
- `datasets/voices/*/Ready/` is currently scaffold-only; no accepted dataset build is tracked.

## 2) Audio requirements

- Input format: common audio formats are accepted by `scripts/build_dataset_from_audio.py`
  (`.wav`, `.mp3`, `.m4a`, `.flac`, `.ogg`, `.aac`, `.opus`, `.wma`, `.aiff`).
- Manifest audio format: generated training chunks are WAV.
- Channels: mono preferred
- No clipping/distortion
- Consistent loudness and low background noise
- Avoid duplicated or near-duplicated samples
- No music, music-like intros/outros, or sustained non-voice content must pass into accepted chunks.
- In `build_dataset_from_audio.py`, non-voice filtering is enabled by default with
  `--voice_filter_mode silero`.
- The default filter path uses local Silero VAD first and rejects uncertain non-speech instead of passing
  continuous music/noise as voice.
- Use `--voice_filter_mode off` to keep legacy behavior when you want zero filtering drift for existing datasets.

### Accepted segment purity rules (applies when filtering is on)
- Segment must satisfy configured speech-overlap purity (`--voice_filter_min_coverage`).
- Segments with low overlap are rejected with `non_voice_ratio_too_high` in report.
- In strict mode (`--strict_mode` or `--voice_filter_mode strict`), default coverage is `1.0` unless an explicit override is provided.
- Dataset manifest schema remains unchanged (`audio`, `text`, `ref_audio`), only filtering behavior changes.

---

## 3) Text requirements

- Text must match spoken content closely.
- Keep punctuation (helps prosody).
- Remove obvious OCR/ASR garbage tokens.
- No empty strings.

---

## 4) Splits and files

Store manifests in:

```text
experiments/qwen3_ru_en_speaker_v1/manifests/
```

Required files for first cycle:
- `train_raw.jsonl` (manual/source manifest)
- `train_with_codes.jsonl` (generated by `prepare_data.py`)

Current state:
- `train_raw.template.jsonl` exists as a schema example.
- Real `train_raw.jsonl` and `train_with_codes.jsonl` are not present in the repository.
- Generated manifests may be committed only when they are small, reproducible, and do not expose private/source paths that should remain local.

Optional:
- `val_raw.jsonl`

---

## 5) Validation checklist

Before preprocessing:
1. Every row is valid JSON.
2. `audio`, `text`, `ref_audio` keys exist.
3. Paths exist and point to files.
4. Text is non-empty after trimming.
5. `audio` paths are unique.

Use:

```bash
source .venv/bin/activate
python scripts/validate_manifest.py --input_jsonl experiments/qwen3_ru_en_speaker_v1/manifests/train_raw.jsonl
```

---

## 6) Handoff checklist

When dataset is provided, include:
- dataset root path,
- single chosen `ref_audio` path,
- estimated hours and number of clips,
- notes about known noisy/low-quality segments.
- confirmation that source audio remains outside Git or under ignored `Input/`.
